# audit-bug

Analiza el bug actual en modo estabilización.

Objetivo:
- NO editar código todavía.
- Encontrar causa raíz probable.
- Enumerar archivos exactos a tocar.
- Proponer fix mínimo.
- Listar riesgos de regresión.

Proceso:
1. Resume el flujo actual relacionado con el bug.
2. Identifica la causa raíz más probable.
3. Enumera exactamente los archivos candidatos.
4. Propón el fix mínimo.
5. Lista checklist de no-regresión.
6. Entrega pasos concretos de validación en Unity.

Restricciones:
- No refactor.
- No renombres.
- No más de 3 archivos sin detenerte y pedir confirmación.
- No tocar escenas/prefabs/ScriptableObjects salvo necesidad explícita y justificada.

This command will be available in chat with /audit-bug
